프로젝트 개요

배경 및 목적

간병인 매칭 플랫폼 '케어네이션'에서는 간병을 필요로 하는 유저가 구인 공고를 올리면 간병인이 원하는 금액으로 지원하는 자유 입찰 방식으로 운영됩니다.
이때, 유저들의 선택을 용이하게 하기 위해, 환자 상태에 따른 적절한 간병비를 계산하여 제시하는 기능을 도입해왔습니다.

하지만 기존 적정 간병비 계산 모델이 실제 매칭 가격과 간극을 나타내게 되었고, 서비스 또한 개편되면서 적정 간병비에 고려할 수 있는 특성들이 추가되었습니다. 또한 시장 금액이 지속적으로 상승하여, 이에 대한 가이드라인을 다시 제시할 필요가 있었습니다.

프로젝트의 목표는 크게 두 가지였습니다:
첫째, 새롭게 추가된 특성들을 고려하여 실제 매칭 가격에 근접한 금액을 예측하는 모델을 개발하되,
둘째, 환자의 어떤 특성이 예측값에 영향을 미치는지를 직관적으로 확인 가능하게 만들기.

모델 개발 과정

1. 데이터 전처리 및 특성 엔지니어링

본격적인 분석에 앞서 데이터 전처리를 진행했습니다.

전처리 과정:

  • 이상치 탐지: IQR 및 고객 응대 자료 기반 이상치 제거
  • 특성 타입 변환: 텍스트 데이터를 범주형 또는 Boolean 형태로 변환
  • 특성 스케일링: StandardScaler를 활용한 정규화
  • 공선성 검증: VIF(Variance Inflation Factor) 분석을 활용해 특성 삭제 및 통합

2. OLS Regression 모델 개발

다양한 특성 조합을 테스트하여 최적의 OLS Regression 모델을 개발했습니다.

모델 개발 과정:

  • 특성 선택: Stepwise selection
  • 모델 학습: OLS Regression 파라미터 최적화
  • 성능 평가: RMSE, R² 등 다양한 지표 활용
  • 교차 검증: K-fold cross validation으로 모델 안정성 검증

3. 모델 단순화 및 서비스 적용

실제 서비스에도 손쉽게 적용 가능하고 직관적으로도 이해 가능하도록
모델의 회귀계수를 기반으로 특성별 가격을 가격 단위로 변환하는 자료를 만들어 전달했습니다.

적용 과정:

  • 모델 단순화: 복잡한 특성 조합을 단순한 형태로 변환
  • 자료 정리: 엑셀에 정리 자료를 만들어서 기획팀 및 개발팀에 전달

주요 학습 내용

  • 선형 회귀 모델링: OLS Regression의 이해와 최적화
  • 특성 선택: 공선성 검증 및 특성 중요도 분석
  • 비즈니스 적용: 머신러닝 모델의 실제 서비스 적용

향후 개선 방향

  • 실시간 시장 데이터 반영을 위한 모델 업데이트 자동화
