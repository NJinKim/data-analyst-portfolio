프로젝트 개요

배경 및 목적

API 통신 오류, 고객 대응팀의 휴먼 에러, 유저의 실수 등으로 과도하게 높거나 낮은 금액이 결제되는 일이 발생했습니다.
이러한 이상 결제는 회사의 재정적 손실과 유저의 이탈을 초래할 수 있는 심각한 문제였습니다.

근무 기간, 유저의 관계성, 결제 금액, 환자 상태 등을 기반으로 결제 금액의 이상치를 탐지할 수 있는 모델을 개발하여
즉각적인 대처를 통해 큰 손실 및 유저 이탈을 방지하는 것이 목표였습니다.

모델 개발 전략

모델 선택 전략

실시간 데이터 탐지에 사용하기 위해 비교적 가벼운 모델을 사용하고자 하였고, 모델 성능 비교를 위해 Gaussian Mixture 모델부터 Isolation Forest, RRCF(Robust Random Cut Forest) 모델 등을 테스트하였습니다.

다중 모델 접근법

  • Gausian Mixture Model: 확률 기반의 군집화 이상치 탐지
  • Isolation Forest: 트리 기반의 효율적인 이상치 탐지
  • RRCF (Robust Random Cut Forest): 스트리밍 데이터에 적합한 이상치 탐지

구현 내용

1. 데이터 전처리 및 특성 분석

결제 데이터의 특성을 분석하고 이상치 탐지에 적합한 특성을 선별했습니다.

전처리 과정:

  • 결측치 처리: 평균값 대체 및 삭제
  • 특성 정규화: MinMaxScaler를 활용한 정규화

2. 다중 모델 개발 및 비교

다양한 이상치 탐지 모델을 개발하고 성능을 비교하여 최적의 조합을 찾았습니다.

여러 모델을 제작한 후 성능 테스트:

  • Gaussian Mixture Model: 확률 기반 군집 이상치 탐지
  • Isolation Forest: 트리 기반 비지도 학습 이상치 탐지
  • RRCF Model: 스트리밍 데이터에 최적화된 이상치 탐지

3. 모델 결과 시각화

세 가지 모델을 matplotlib 및 plotly로 시각화하여 보고 자료를 구성하였습니다.
이 과정에서 세 모델에서 모두 공통으로 이상치로 탐지된 결제건을 확인할 수 있었고
실시간 데이터 탐지를 도입하는 대신 이상치 탐지로 식별된 데이터를 기반으로 고객 관리 팀에서 결제 승인 금액을 관리하기로 결정하였습니다.
